# Student Expense Tracker (UniBudget)

UniBudget is a student-focused expense tracking web interface designed to help university students manage their finances in a simple, clean, and interactive way. The project was created for User Experience Design and focuses on helping students quickly record expenses, monitor spending, view financial breakdowns, and build better budgeting habits.

## Overview

University students often manage money on limited budgets from part-time jobs, scholarships, or savings. Many finance apps feel cluttered or are not designed around student needs. UniBudget was designed to solve that problem by offering a more intuitive interface with clear visual feedback, simple navigation, and student-centered budgeting features.

## Features

- **Dashboard overview** for a quick summary of finances
- **Net Balance card** showing how much is safe to spend
- **Expense tracking** to log and manage expenses
- **Income tracking** to record money coming in
- **Budget page** to track limits and category budgets
- **University section** for student information and university-related expenses
- **Quick Add modal** for fast transaction entry
- **Search functionality** for finding transactions quickly
- **Time filters** such as “This Week” and “This Month”
- **Hover effects and interactive feedback** for better usability
- **Visual breakdowns** to help students understand spending patterns

## Design Goals

This project was built around a user-centered design approach. The main design goals were:

- Make expense logging fast and easy
- Present financial information clearly
- Keep the interface visually clean and modern
- Help students understand their spending at a glance

The final design uses:

- **Indigo accent color** `#4F46E5`
- **Green** to represent income
- **Red** to represent expenses
- Rounded corners and a clean white layout for a modern look

## Design Process

The project followed a structured UI design workflow:

1. **Problem Identification & User Research**  
   We identified the need for a finance tracker tailored to students and interviewed university students to better understand their pain points.

2. **Wireframing**  
   We began with low-fidelity sketches and grid-based layouts to establish spacing, structure, and navigation.

3. **High-Fidelity Prototyping**  
   We developed the UniBudget visual identity and refined the layout into a polished interface.

4. **Evaluation & Feedback**  
   We applied heuristic evaluation and user testing to identify usability issues and gather feedback.

5. **Refinement**  
   Based on user feedback, we improved icon clarity and made budget information more prominent on the dashboard.

## Usability Evaluation

The interface was evaluated using usability principles and user testing.

### Heuristic Evaluation
The design considered Nielsen’s heuristics, with emphasis on:

- Visibility of system status
- Consistency and standards

Early evaluation highlighted issues such as unclear icons and labeling, which were improved in later revisions.

### User Testing
Participants completed tasks such as:

- Adding expenses
- Setting budgets
- Navigating between pages

Feedback was generally positive, especially around ease of use and clarity, while some users suggested improving category icons and emphasizing important budget information.
